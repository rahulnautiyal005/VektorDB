# 🌌 VektorDB

**Hardware-Accelerated, Out-of-Core Vector Search Engine for AI Workloads**

*C++20 | SIMD (AVX-512) | HNSW | mmap | gRPC*

---

💡 **Engineered for sub-millisecond similarity search across billion-scale vector datasets, bypassing RAM limits via memory-mapped I/O.**

---

## 📌 Architecture Overview

**VektorDB** is a native C++ vector database designed to act as the retrieval backend for Large Language Models (RAG).

It implements the **Hierarchical Navigable Small World (HNSW)** algorithm for Approximate Nearest Neighbor (ANN) search, optimized with CPU-specific SIMD instructions for lightning-fast distance calculations.

---

## 🧠 Core Engineering Achievements

### ⚡ SIMD-Accelerated Math

* Hand-written **AVX2 / AVX-512 intrinsics** for:

  * Cosine Similarity
  * L2 (Euclidean) Distance
* Achieves **8x–16x throughput improvement** over naive implementations

---

### 💾 Out-of-Core Storage

* Zero-copy memory-mapped files using **mmap**
* Uses **madvise** for optimized OS-level paging
* Enables search on datasets **larger than RAM (NVMe-backed)**

---

### 🔗 Concurrent HNSW Graph

* Lock-free / fine-grained locking design
* High-throughput concurrent reads
* Thread-safe batch inserts

---

### 🐍 Python Interoperability

* Zero-overhead bindings using **pybind11**
* Acts as a drop-in replacement for:

  * FAISS
  * Annoy

## 🛣️ Implementation Roadmap

> ⚠️ **This is a marathon, not a sprint. Build from CPU → Storage → Index → API.**

---

### 🧮 Phase 1: The Math Engine (Weeks 1–2)

Goal: Compute vector similarity as fast as possible.

* Implement baseline:

  * Euclidean Distance (L2)
  * Cosine Similarity

    ```
    cos(θ) = (A · B) / (||A|| ||B||)
    ```

* Setup **Google Benchmark** for performance measurement

* Rewrite using **AVX2 intrinsics** (e.g., `_mm256_dp_ps`)

* Compare performance → target **massive latency reduction**

💡 *First major interview talking point*

---

### 💾 Phase 2: Flat Search & Memory Mapping (Weeks 3–4)

* Design custom `.vkdb` binary format:

  * Header (metadata, dimension, vector count)
  * Contiguous float vectors

* Implement storage using **mmap**

* Build naive **O(N)** nearest neighbor search

* Use SIMD math for fast computation

---

### 🧠 Phase 3: HNSW Graph (Weeks 5–7) ⚠️ *Hardest Part*

* Study original HNSW paper (Malkov & Yashunin)

#### Key Components:

* Multi-layer graph structure
* Adjacency list optimized for memory locality
* Greedy search algorithm
* Probabilistic insertion logic

---

### ⚙️ Phase 4: Concurrency & API (Weeks 8–9)

* Add thread safety:

  * `std::shared_mutex` for read-write locks

* Build **gRPC server**

  ```
  service Vektor {
    rpc Search(Query) returns (Results);
  }
  ```

* (Optional 🔥) Add Python bindings using **pybind11**
